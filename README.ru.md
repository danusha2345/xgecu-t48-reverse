# xgecu-t48-reverse

Реверс USB-протокола и Python-прототип для работы с универсальным
программатором **XGecu T48 (TL866-3G)** — с акцентом на чтение/запись
**eMMC**, в том числе внутрисхемно (ISP).

Программатор XGecu T48 — популярное универсальное устройство; его
первородный софт (`Xgpro`) только под Windows, а USB-протокол не
документирован. Открытый проект [`minipro`](https://gitlab.com/DavidGriffith/minipro)
покрывает «классическую» часть протокола, но **намеренно не
реализует** eMMC. Этот репозиторий документирует eMMC-расширение
протокола — как реально выглядит проводной формат — и содержит
небольшой Python-прототип, который использует тот же encoding.

> **Статус: до приезда железа.** Протокол вытащен статическим анализом
> `Xgpro.exe` (PE32, WinUSB). Все три команды, расшифрованные прямо из
> бинаря (CMD6 SWITCH → RPMB, SWITCH → USER, SWITCH → HS-200),
> воспроизводятся **побайтово**. То, что требует живой проверки
> (детали init-handshake, форматы ответов, тайминги), помечено как TBD
> и будет проверено USB-захватом, когда программатор приедет.

## Что внутри

- [`docs/PROTOCOL.md`](docs/PROTOCOL.md) — полная техдокументация
  реверса: USB-идентификация (`a466:0a53`, WinUSB), роли endpoint'ов
  (EP1 OUT/IN для команд + EP2 OUT для bulk), иерархия
  top-opcode / sub-opcode (унифицирована с minipro), таблица eMMC-
  команд, init-последовательность, layout JEDEC RPMB-frame, формат
  файла `.alg` и форматы bulk-передач. Русская версия:
  [`PROTOCOL.ru.md`](docs/PROTOCOL.ru.md).
- [`examples/t48_emmc.py`](examples/t48_emmc.py) — небольшой
  pyusb-прототип: подключается к T48 по VID:PID, собирает
  документированные пакеты и вызывает их через высокоуровневые методы
  (`switch_partition`, `read_ecsd`, `bulk_read`, билдер RPMB-frame и
  т.д.). Включает offline-тест, проверяющий, что три расшифрованные
  CMD6 SWITCH-команды совпадают с байтами из `Xgpro.exe`. Любой
  USB-трансфер можно логировать в файл (`T48Emmc(log_path=…)`), чтобы
  реальная сессия стала «эталоном» для протокол-дока.
- [`examples/first_contact.py`](examples/first_contact.py) —
  **безрисковый** первый прогон, который запускают сразу после
  подключения T48: `connect → identify → измерение напряжений → status
  → чтение пинов`. Не подаёт программирующее напряжение, не требует ни
  чипа, ни eMMC-ISP адаптера, и логирует каждый трансфер. Безопасный
  способ проверить транспорт и классические опкоды до того, как трогать
  чип.
- [`tools/extract_alg.py`](tools/extract_alg.py) — самостоятельный
  распаковщик проприетарных `.alg`-файлов из `Xgpro`. Каждый `.alg`
  содержит zero-RLE-сжатый битстрим Xilinx Spartan-6 FPGA (340 604
  байта в распакованном виде) с 32-битным CRC. Сами `.alg` файлы НЕ
  входят в репозиторий — приноси из своей легальной установки Xgpro.

## Беглый взгляд на протокол

Любая USB-команда `Xgpro` на EP1 OUT начинается с однобайтового
**top-opcode**. Те, что в классическом диапазоне (`0x02..0x3F`), уже
реализованы в `minipro`. eMMC-расширение добавляет такие top-opcode'ы:

| Top-opcode | Назначение                                                |
|------------|-----------------------------------------------------------|
| `0x08`     | Обёртка с большим recv (sub-op `0x48` → read 512 байт)    |
| `0x14`     | Bulk-write setup перед N × 512 байт на EP2 OUT            |
| `0x21`     | eMMC init / выбор алгоритма                               |
| `0x27`     | Диспетчер eMMC sub-команд (sub-op в байте 1)              |

Для команд класса `0x27` пакет 8 байт:
`[0x27][sub-op][u16 = 0][u32 arg LE]`. Среди sub-op'ов самое
информативное — кодировка **CMD6 SWITCH**: 32-битный `arg` — это
ровно аргумент JEDEC eMMC CMD6 в big-endian, поэтому, например:

```
SWITCH PARTITION_CONFIG → RPMB     →  arg = 0x01B30300
SWITCH HS_TIMING        → HS-200   →  arg = 0x01AF0100
restore PARTITION_ACCESS → USER    →  arg = 0x02B30700  (CLEAR_BITS со значением 0x07)
```

Полная таблица — в `docs/PROTOCOL.md` §7.

## Запустить прототип offline

```bash
pip install pyusb
python3 examples/t48_emmc.py
```

Это прогонит байтовый sanity-тест без обращения к железу:

```
== Проверка структур пакетов (offline) ==
  cmd_A(SWITCH→RPMB)        = 274600000003b301
  cmd_A(SWITCH→USER restore)= 274600000007b302
  cmd_A(SWITCH→HS200)       = 274600000001af01
  ...
[OK] Все три расшифрованные команды сходятся с реверсом Xgpro.exe.
```

С подключённым T48 команда `python3 examples/t48_emmc.py --connect`
попытается открыть устройство через libusb. На Linux может
понадобиться udev-правило, чтобы дать твоему пользователю доступ к
интерфейсу; на Windows подходит официальный WinUSB-драйвер от XGecu.

Для **первого** прогона на свежем устройстве лучше использовать
безопасный пробник — он не подаёт напряжение, не требует чипа и
сохраняет полный лог трансферов:

```bash
python3 examples/first_contact.py          # → t48_first_contact.log
```

## Распаковка `.alg` (опционально)

Если нужно посмотреть FPGA-битстрим, который Xgpro поставляет для
конкретного семейства чипов, распаковщик пишет сырой 340604-байтный
payload:

```bash
python3 tools/extract_alg.py path/to/Xgpro/algorithm/EMMC_41_18.alg
# → EMMC_41_18.bit  (340604 байт, 16-байт 0xFF преамбула + FPGA sync)
```

## Чего здесь *нет*

По задаче проекта репозиторий содержит только оригинальные материалы —
техдокументацию реверса и наш собственный код, *говорящий
документированным протоколом*. Сюда специально **не входят**:

- никакие бинарники XGecu (`Xgpro.exe`, драйверы, прошивки);
- никакие `.alg`-файлы;
- дамп БД чипов (`InfoIC2Plus.dll`);
- распакованные FPGA-битстримы.

Это копирайтные материалы XGecu. Софт `Xgpro` свободно скачивается у
производителя (https://www.xgecu.com/), а утилита `dumpic` из `minipro`
делает дамп БД чипов локально из легальной установки.

## Зачем это нужно

Изначальный потребитель материала — отдельное eMMC-приложение, которое
уже поддерживает другие USB-eMMC backend'ы и хочет добавить T48 как
ещё один backend под Windows и Linux. Публиковать документацию
протокола полезно ещё и для всех, кто запускает T48 из Linux и хочет
что-то более программируемое, чем запуск `Xgpro` под Wine.

Работа надстраивает прекрасный [`minipro`](https://gitlab.com/DavidGriffith/minipro)
(GPL-3.0) — он уже реализует общий транспорт T48 (опкоды
`0x02..0x3F`, USB-handshake, управление напряжением/частотой) — и
[`radiomanV/TL866`](https://github.com/radiomanV/TL866) с его реверсом
прошивки этого семейства устройств.
