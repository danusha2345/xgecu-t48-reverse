# XGecu T48 — справочник по настройкам eMMC ISP

Что делает каждая настройка Xgpro для **внутрисхемной (ISP) работы с
eMMC** и что включать/выключать. Собрано из трёх источников: строки и
логика установленного `Xgpro.exe`, официальный мануал Xgpro, и наши
захваты на уровне провода (`docs/PROTOCOL.md` §33–§34 — они показывают,
что каждый выбор реально делает в USB-протоколе).

Короткие цитаты ниже (`"..."`) — дословно из `Xgpro.exe`, оставлены для
совместимости.

---

## 1. Выбор IC — `AUTO_EMMC(ISP)_<width>bit_<VCCQ>` (главная настройка)

Выбранный IC кодирует **сразу две** вещи (подтверждено в захватах):

- **Ширина шины** (1 / 4 / 8-бит). Это host-side выбор: задаёт FPGA-
  битстрим и проявляется одним байтом в `BEGIN_TRANSACTION`
  (`0x51` = 1-бит, `0x54` = 4-бит; §33.6). CMD6 `BUS_WIDTH`-свитч чипу
  **не** шлётся — программатор/адаптер сам гонит шину в выбранном режиме.
  - **1-бит** — минимум проводов, максимально терпимо к корявому ISP-
    монтажу. *Начинай отсюда.*
  - **4 / 8-бит** — быстрее, но нужно больше чистых линий данных и хороший
    сигнал.
- **VCCQ (I/O напряжение)** — `1.8V` или `3.3V` (см. §4).

## 2. Интерфейс: `ICSP port` vs `ZIF/SIP ADP`

- **ICSP port** — внутрисхемно, через ISP-разъём. **Для ISP eMMC всегда
  это.** Xgpro подсказывает: *"Connect <ICSP> to target board"*,
  *"Guaranteed: the ICSP connection is correct!"*.
- **ZIF/SIP ADP** — сокет на корпусе, для выпаянных чипов на BGA-адаптере.
  Не для внутрисхемной работы.

## 3. `ICSP_VCC Enable` — кто питает цель (самый неоднозначный тумблер)

- **Включено** → T48 сам подаёт VCC на цель по ISP-кабелю, **макс
  120 мА** (*"Option VCC power (MAX120ma) to the target board"*). Когда
  плата обесточена и хочешь запитать только чип от программатора.
- **Выключено** → цель питается снаружи. Тогда `Xgpro.exe` прямо говорит:
  **"If use external power don't connect VCC"** — линию VCC физически НЕ
  подключать, чтобы два источника не конфликтовали.

Правило: питаешь чип от T48 → *включить*; плата уже под питанием →
*выключить* и не цеплять VCC.

## 4. VCCQ — `1.8V` vs `3.3V`

- Большинство eMMC: `VCC = 3.3V / VCCQ = 1.8V` (мануал отмечает высокую
  стабильность).
- Многие читаются и на 3.3V. `Xgpro.exe` для части чипов предупреждает:
  *"EMMC not Support 1.8V Low Voltage Supply"*, а при сбое чтения:
  **"Reduce the CLK or switch VCCQ 1.8V<->3.3V for testing"**.
- Выбирай уровень, на который рассчитан чип; при сбоях — переключи VCCQ.

## 5. `CLK` — частота шины

- Опции: **`AUTO (MAX = 50 MHZ)`** плюс вручную 8…50 МГц и HS-режимы
  (60/80 = HS200, 100/120/160 = HS400). Через ISP практический потолок
  ~40–50 МГц.
- **Оставь AUTO.** Если чтение нестабильно — **снижай CLK** (первый шаг
  диагностики у Xgpro). Длинные/неаккуратные ISP-провода ⇒ ниже частота.

## 6. `Vcc current Imax`

- Лимит тока (защита от КЗ). **`Default`** в обычном случае; уменьшить —
  для доп. защиты на незнакомой плате (сработает раньше).

## 7. Разделы (Partition)

Переключение разделов — это CMD6 `SWITCH` в `PARTITION_CONFIG[179]`,
подтверждено на проводе (§33.3):

| Раздел | Значение PARTITION_CONFIG |
|--------|---------------------------|
| USER (основной) | clear (`07 b3 02`) |
| BOOT1 | `01 b3 01` |
| BOOT2 | `02 b3 01` |
| RPMB  | `03 b3 01` |
| GPP1–4 | general-purpose разделы |

- Выбираешь, какие разделы читать / писать / стирать
  (*"Erase / Blank Check / Partition Selected"*). Полный клон читает все
  системные разделы + USER.
- **RPMB** требует ключ аутентификации (HMAC). Без него:
  *"Authentication Key not yet programmed, RPMB not used"*.

## 8. Аппаратное (не софт, но обязательно для ISP)

- **RST_n**: после подачи питания `RST_n` eMMC должен быть высоким. Если
  читается 0 — подтяни к VCCQ резистором **~1 кОм**, иначе чип в сбросе и
  не ответит.
- **Земли**: у ISP две GND — подключать **обе**, возврат земли **близко к
  линии CLK**. CLK не пересекать с другими сигналами.
- При нестабильности помогает конденсатор **2.2 мкФ** между VCC и GND
  (*"Connect a 2.2uf Capacitor between Pin VCC and Gnd"*).

## 9. TL;DR — стабильная отправная точка для ISP-чтения

1. IC: **`AUTO_EMMC(ISP)_1bit`** (на 4-бит — когда монтаж проверен);
   VCCQ по чипу.
2. Интерфейс: **ICSP port**.
3. **ICSP_VCC Enable**: вкл, если питаешь чип от T48 (≤120 мА); выкл и не
   цепляй VCC, если плата питается снаружи.
4. CLK: **AUTO**; при ошибках — ниже.
5. Imax: **Default**.
6. Железо: RST_n подтянут к VCCQ (~1 кОм), обе GND у CLK.

Замечание: ISP-адаптер сам аутентифицируется к прошивке T48 (по USB он
представляется как `XGecu Directly` через opcode `0x24`, §33.1); ничего из
вышеперечисленного не является крипто-настройкой — это выбор
сигналов/электрики и разделов.
